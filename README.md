# Countdown Bot

Telegram-бот, который каждый день обновляет закреплённое сообщение в одном
конкретном чате текстом вида:

> **Чемпионат Казахстана**
> Осталось 87 дней

---

## 1. Создать бота в Telegram

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → придумай имя.
2. Скопируй токен (выглядит как `123456:ABC...`).
3. У того же BotFather: `/setprivacy` → выбери своего бота → **Disable**
   (не обязательно, но удобно — пусть видит команды в группах).

## 2. Узнать свой `ADMIN_ID`

Напиши боту [@userinfobot](https://t.me/userinfobot) — он пришлёт твой
числовой `id`. Это и есть `ADMIN_ID`.

## 3. Добавить бота в нужный чат

1. Добавь созданного бота в чат, где должен висеть закреп.
2. Сделай его **администратором** с правом **«Закреплять сообщения»**.
3. В этом чате отправь `/start@твой_бот` — он покажет ID этого чата.

## 4. Локальный запуск (для проверки)

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

export BOT_TOKEN="123456:ABC..."
export ADMIN_ID="123456789"
export TZ="Asia/Almaty"

python bot.py
```

Затем в личке боту:

```
/setup 2026-09-01 Чемпионат Казахстана
```

— бот отправит и закрепит сообщение в чате, куда его добавили.

## 5. Деплой на Railway (бесплатный план)

1. Залей этот проект в репозиторий на GitHub.
2. Зайди на [railway.app](https://railway.app) → **New Project** →
   **Deploy from GitHub repo** → выбери репу.
3. В разделе **Variables** добавь:
   - `BOT_TOKEN` = токен от BotFather
   - `ADMIN_ID` = твой Telegram user id
   - `TZ` = `Asia/Almaty`
   - (опционально) `UPDATE_HOUR` = `9`, `UPDATE_MINUTE` = `0`
4. Railway сам поймёт `Procfile` и запустит `worker: python bot.py`.

### Альтернатива: Render

1. [render.com](https://render.com) → **New** → **Background Worker**.
2. Подключи репу, build command: `pip install -r requirements.txt`,
   start command: `python bot.py`.
3. Добавь те же переменные окружения.

> ⚠️ **Важно про state.json.** Файл с настройками лежит на диске сервиса.
> На Railway/Render бесплатный план может пересоздавать контейнер —
> при этом `state.json` обнуляется и нужно снова вызвать `/setup`.
> Если хочется надёжности, подключи постоянный диск (Volume на Railway,
> Disk на Render) и поставь `STATE_FILE=/data/state.json`.

## Команды бота

| Команда | Что делает |
|---|---|
| `/start` | Показывает справку и твой `user_id` / ID чата |
| `/setup YYYY-MM-DD Название` | Задаёт дату и название (только админ) |
| `/show` | Показывает текущую настройку и сразу обновляет закреп |
| `/stop` | Очищает настройку, обновления прекращаются |

## Как это работает

- При старте и затем **каждый день в 09:00** (по `TZ`) бот:
  1. Считает, сколько дней осталось до целевой даты.
  2. Отправляет новое сообщение в чат.
  3. Открепляет старое (если оно от него), закрепляет новое без уведомления.
- Состояние (chat_id, дата, название, id последнего закрепа) хранится
  в `state.json`.
